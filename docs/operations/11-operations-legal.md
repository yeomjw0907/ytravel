# Ytravel Operations and Legal Guardrails

## 1. Purpose

이 문서는 Ytravel의 운영 리스크, 서비스 문구 기준, 가격 정보 고지 방식, 링크아웃 안내, BRG 관련 주의사항을 정리한다.

## 2. Service definition

- Ytravel은 예약, 결제, 환불을 처리하는 서비스가 아니다.
- Ytravel은 사용자가 입력한 예약 조건을 기준으로 더 저렴한 후보를 탐색하고 비교 정보를 보여주는 서비스다.
- 결과는 참고 정보이며, 최종 예약 조건과 결제 금액은 외부 사이트에서 다시 확인해야 한다.

## 3. Pricing disclosure rules

- 가격은 수집 시점 기준 정보임을 항상 명시한다.
- 세금, 봉사료, 리조트피, 프로모션, 결제 방식에 따라 총액이 달라질 수 있음을 안내한다.
- 카드에 보이는 가격과 실제 결제 가격이 다를 수 있음을 결과 화면 하단에 고정 노출한다.
- `collectedAt`은 검색 결과와 상세 화면에 표시한다.

## 4. BRG disclosure rules

- Ytravel은 BRG 승인 여부를 보장하지 않는다.
- BRG는 호텔 브랜드의 정책, 제출 시점, 공개 요금 조건, 취소 정책에 따라 달라질 수 있다.
- 결과 화면의 BRG 관련 문구는 모두 `참고`, `가능성`, `직접 확인 필요` 톤으로 유지한다.
- `보장`, `확정`, `무조건`, `100%` 같은 표현은 금지한다.

## 5. Provider and crawling guardrails

- 승인되지 않은 공급처에 대해 공개 웹 크롤링을 기본 전략으로 삼지 않는다.
- 자동 수집이 불안정한 공급처는 `reference` 또는 `link_only` 상태로만 노출한다.
- 공급처 구조 변경, 차단 징후, 가격 오차가 반복되면 즉시 비활성화한다.
- robots, 이용약관, 파트너 정책은 정기적으로 확인한다.

## 6. UI copy requirements

### Search/result footer message

`표시된 가격은 수집 시점 기준 참고 정보이며 실제 예약 화면과 다를 수 있습니다. 최종 금액과 조건은 외부 사이트에서 다시 확인해 주세요.`

### Reference mode message

`이번 결과는 실시간 확정가가 아니라 동일 조건 재확인을 위한 참고 후보입니다.`

### External link message

`외부 사이트에서 객실 조건, 취소 규정, 조식 포함 여부, 세금 포함 여부를 다시 확인해 주세요.`

## 7. Contact and incident handling

- 문의 채널은 최소 1개를 항상 공개한다.
- 가격 오류 신고가 들어오면 수집 시각, 공급처, 통화, 세금 포함 여부부터 확인한다.
- 외부 링크 오류가 반복되면 해당 공급처를 임시 비활성화한다.
- 법무 이슈나 삭제 요청이 오면 관련 노출을 먼저 중단하고 내부 검토 후 복구한다.

## 8. Launch checklist

- 모든 법무/안내 페이지가 실제 라우트로 동작한다.
- 홈, 검색, 상세, 푸터 문구가 동일한 톤을 유지한다.
- `reference`와 `live` 상태가 화면에서 명확히 구분된다.
- 외부 링크 안내가 모든 relevant CTA 근처에 노출된다.
- 문의 채널이 실제로 동작한다.

## 9. Monitoring

- 공급처별 fetch 성공률
- 공급처별 응답 실패율
- 검색 대비 외부 링크 클릭률
- 결과 없음 비율
- 가격 오차 문의 건수

## 10. Launch posture

- 이번 주 기본 원칙은 `정확성 우선`이다.
- 자동화 범위보다 신뢰 가능한 문구와 상태 표기가 더 중요하다.
- 불확실한 결과는 숨기지 말고 `참고 후보`로 명확하게 보여준다.
