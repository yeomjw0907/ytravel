# Ytravel 디자인 가이드

## 1. 디자인 방향

Ytravel은 일반적인 여행 예약 서비스보다 더 차분하고 정제된 인상을 가져야 한다.  
핵심 무드는 `심야의 하늘`, `정제된 신뢰감`, `고급스러운 비교 도구`다.

브랜드가 전달해야 하는 감정

- 복잡한 가격 비교를 대신 정리해주는 신뢰감
- 호텔 공식가와 OTA를 냉정하게 비교하는 정밀함
- 여행 전 설렘보다 예약 판단의 확신에 가까운 분위기

## 2. 비주얼 키워드

- Silent Luxury
- Night Sky
- Premium Utility
- Calm Precision
- Minimal Contrast

## 5. BRG 상태 색상 규칙

Ytravel은 상태색을 과도하게 쓰지 않는다. 기본은 절제된 블루/그레이 톤을 유지하고, 상태 영역에서만 포인트를 준다.

- `BRG 가능성 높음`
  - 배경: `#E7F3EC`
  - 텍스트: `#2D6A4F`
- `조건 검토 필요`
  - 배경: `#F6EEDC`
  - 텍스트: `#8B6508`
- `동일 조건 아님`
  - 배경: `#F3E4E4`
  - 텍스트: `#A63A3A`
- `공식가 기준`
  - 배경: `#EAF1F6`
  - 텍스트: `#04345A`

## 6. 그라디언트 사용 규칙

서비스의 대표 그라디언트는 아래 3개만 사용한다.

### Hero Gradient

```css
linear-gradient(135deg, #050606 0%, #04345A 45%, #06588C 100%)
```

### Surface Gradient

```css
linear-gradient(135deg, #04345A 0%, #626979 100%)
```

### Fog Gradient

```css
linear-gradient(135deg, #C8C6C6 0%, #04345A 100%)
```

규칙

- 한 화면에 강한 그라디언트는 1개만 쓴다.
- 카드 본문에는 그라디언트를 남용하지 않는다.
- CTA, 히어로, 핵심 요약 박스에만 제한적으로 사용한다.

## 7. 타이포그래피 가이드

브랜드 톤은 `여행 서비스`보다 `프리미엄 리서치 툴`에 가깝다.

### 추천 폰트 조합

- Heading: `Cormorant Garamond` 또는 `Playfair Display`
- Body/UI: `Pretendard` 또는 `SUIT`

### 타이포 역할

- Display: 브랜드 카피, 히어로 타이틀
- Heading: 섹션 제목, 카드 제목
- Body: 설명문, 가격 비교 텍스트
- Caption: 조건 태그, 메타 정보

### 텍스트 톤

- 문장은 짧고 단정하게 쓴다.
- 과장된 여행 감성 문구보다 정확한 비교 문구를 우선한다.
- 숫자와 조건 정보는 읽기 쉬운 고정폭에 가까운 정렬감을 유지한다.

## 8. UI 스타일 원칙

### 배경

- 전체 바탕은 밝은 종이 질감 느낌의 `Cloud Paper` 계열
- 섹션은 큰 면 분할로 구분
- 다크 배경은 핵심 섹션에서만 사용

### 카드

- 둥근 모서리 20px~28px
- 얇은 보더 또는 낮은 대비 그림자 사용
- 카드 내부 여백은 넉넉하게
- 정보 밀집형 카드라도 여백으로 고급감을 유지

### 버튼

- Primary Button: Abyss Blue 바탕, 밝은 텍스트
- Secondary Button: 투명 또는 연한 회색 바탕, 짙은 보더
- Ghost Button: 텍스트 중심

버튼 스타일 기준

- 높이 48px 이상
- 좌우 패딩 충분히 확보
- 호버 시 채도 상승보다 명도 변화 위주

### 입력창

- 배경은 밝은 톤 유지
- 포커스 시 `#06588C` 보더 또는 글로우
- 달력/인원 선택 UI도 동일 톤 유지

## 9. 레이아웃 원칙

### 홈

- 히어로가 가장 강해야 한다.
- 검색 폼이 첫 시선에 들어와야 한다.
- 설명 섹션은 3단 이하로 정리한다.

### 검색 결과

- 최상단에 검색 요약
- 그 다음 BRG 요약 박스
- 그 아래 공급처 비교 리스트
- 조건 차이 정보는 가격 바로 아래에서 보여준다.

### 상세 페이지

- 공급처별 가격 비교
- 객실 조건 비교
- BRG 판단 근거
- 외부 링크

순서로 고정한다.

## 10. 컴포넌트 디자인 기준

### BrgSummaryCard

- 배경은 브랜드 그라디언트 또는 깊은 블루 단색
- 공식가, OTA 최저가, 예상 BRG가를 3분할로 배치
- 상태 배지는 우상단 고정

### ProviderRateCard

- 공급처 로고
- 가격
- 조건 태그
- 보기 버튼

가격과 상태가 먼저 보이고, 세부조건은 아래 태그로 읽히게 한다.

### ConditionBadge

- 작은 캡슐 형태
- 조식 포함, 무료 취소, 세금 포함 등 짧은 정보 위주
- 배경은 연한 회색 또는 상태별 연색 사용

## 11. 아이콘 및 이미지 방향

- 선이 얇고 정제된 아이콘 사용
- 여행 사진은 밝은 휴양지 감성보다 `하늘`, `도시 야경`, `호텔 외관`, `밤의 정적` 계열 선호
- 이미지가 없어도 UI가 성립해야 한다.

## 12. 모션 가이드

- 페이지 첫 진입 시 상단 섹션 페이드업
- 검색 결과 카드는 40ms~80ms 간격의 짧은 스태거
- 숫자 변화는 부드러운 카운트업 가능
- 과한 바운스, 탄성 효과는 금지

## 13. 접근성 기준

- 주요 텍스트 대비는 WCAG AA 이상 목표
- 블루 텍스트를 연회색 위에 올릴 때 대비 확인 필수
- 색상만으로 상태를 구분하지 않고 라벨을 함께 표시
- 모바일에서도 가격, 상태, 버튼이 한 줄에 무너지지 않게 설계

## 14. 브랜드 문구 톤

좋은 예시

- `공홈보다 낮은 요금, 같은 조건인지 먼저 확인하세요`
- `Ytravel이 BRG 후보를 정리해드립니다`
- `가격보다 중요한 건 동일 조건 비교입니다`

피해야 할 예시

- `무조건 최저가 보장`
- `100% BRG 성공`
- `가장 싼 가격 확정`
