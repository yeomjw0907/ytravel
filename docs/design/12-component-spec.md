# Ytravel 컴포넌트 상세 명세서

## 1. 목적

이 문서는 Ytravel의 핵심 UI 컴포넌트를 구현 가능한 수준으로 정의한다.

원칙

- 비교 서비스에 맞는 정보 우선순위를 유지한다.
- 한국어 UI 문장 길이를 고려해 레이아웃을 설계한다.
- 고급스럽고 차분한 인상을 유지하되, 가격과 상태는 즉시 읽히게 한다.

## 2. 공통 규칙

### 2.1 네이밍

- 컴포넌트 이름은 역할 중심으로 정의한다.
- 시각 표현보다 도메인 의미를 우선한다.

예시

- `BrgSummaryCard`
- `ProviderRateTable`
- `MatchStatusPill`

### 2.2 한국어 UI 원칙

- 라벨은 짧고 명확하게 쓴다.
- 설명 문구는 단정체로 작성한다.
- 버튼 텍스트는 2자~8자 범위를 권장한다.

예시

- 적절: `가격 보기`, `조건 비교`, `다시 검색`
- 부적절: `지금 바로 최저가 상세정보를 확인해보세요`

### 2.3 정보 위계

컴포넌트 내부 정보는 아래 우선순위를 따른다.

1. 가격
2. 상태
3. 조건
4. 행동 버튼
5. 부가 설명

## 3. SearchBar

### 역할

- 호텔명과 일정, 인원 정보를 입력받아 검색을 시작한다.

### 구성 요소

- 호텔명 입력 필드
- 체크인 필드
- 체크아웃 필드
- 투숙 인원 선택
- 객실 수 선택
- 검색 버튼

### 레이아웃

- 데스크탑: 한 줄 배치
- 태블릿: 2행 허용
- 모바일: 세로 스택

### 필드 라벨

- 호텔명
- 체크인
- 체크아웃
- 투숙 인원
- 객실 수

### Placeholder 가이드

- 호텔명: `예: 그랜드 하얏트 서울`
- 체크인/체크아웃: placeholder 최소화, 달력 선택 우선

### Validation 메시지

- `호텔명을 입력해 주세요`
- `체크아웃 날짜를 다시 확인해 주세요`
- `투숙 인원은 1명 이상이어야 합니다`

## 4. BrgSummaryCard

### 역할

- 검색 결과의 핵심 결론을 요약한다.

### 필수 정보

- 호텔명
- 공식 홈페이지 총액
- OTA 최저가
- 예상 BRG가
- BRG 상태
- 수집 시각

### 정보 배치

- 상단 좌측: 호텔명
- 상단 우측: 상태 배지
- 중단: 가격 3분할
- 하단: 보조 설명 또는 안내 문구

### 가격 라벨

- `공식 홈페이지`
- `OTA 최저가`
- `예상 BRG가`

### 상태 문구

- `가능성 높음`
- `검토 필요`
- `조건 불일치`
- `데이터 부족`

### 보조 문구 예시

- `현재 수집 기준으로 OTA 최저가가 더 낮습니다`
- `동일 조건 여부를 확인한 뒤 BRG를 검토하세요`

## 5. ProviderRateTable

### 역할

- 공급처별 요금을 표 형식으로 비교한다.

### 컬럼 구조

- 공급처
- 총액
- 취소
- 조식
- 세금
- 결제
- 보기

### 정렬 규칙

- 기본: 총액 오름차순
- 공식 홈페이지는 가격 순서와 무관하게 상단 고정 가능

### 셀 규칙

- 가격 셀은 우측 정렬
- 상태성 텍스트는 캡슐형 태그 사용
- 액션 셀은 `보기` 또는 `이동` 버튼 사용

### 한국어 헤더 원칙

- 짧고 명사형 사용
- 예: `총액`, `조식`, `취소`, `세금`

## 6. ProviderRateCard

### 역할

- 모바일 및 좁은 화면에서 공급처별 가격을 카드 형태로 표현한다.

### 필수 정보

- 공급처명
- 가격
- 최저가 배지 여부
- 취소/조식/세금 태그
- 상세 또는 외부 이동 버튼

### 카드 구조

- 상단: 공급처명 + 가격
- 중단: 상태 태그
- 하단: 버튼

### 배지 종류

- `최저가`
- `공식가`
- `수집 실패`

## 7. MatchStatusPill

### 역할

- 동일 조건 비교 수준을 표현한다.

### 상태값

- `완전 일치`
- `유사 조건`
- `검토 필요`
- `비교 어려움`

### 사용 위치

- BRG 요약 카드
- 공급처 카드
- 상세 비교 표 상단

## 8. ConditionBadge

### 역할

- 객실 조건을 짧게 요약한다.

### 사용 가능한 라벨

- `무료 취소`
- `조식 포함`
- `세금 포함`
- `선결제`
- `현장 결제`
- `객실명 유사`

### 규칙

- 한 카드에서 4개를 넘기면 줄바꿈 허용
- 긴 문장은 금지
- 상태 의미가 중복되면 제거

## 9. PriceDelta

### 역할

- 공식가 대비 차액과 할인 폭을 표현한다.

### 표시 방식

- `공식가 대비 52,000원 낮음`
- `16.2% 차이`

### 규칙

- 절대 금액 우선, 퍼센트는 보조 정보
- 차액이 작아도 과장 표현 금지

## 10. HotelHeader

### 역할

- 호텔 페이지 상단에서 기본 정보를 정리한다.

### 필수 정보

- 호텔명
- 도시/국가
- 브랜드
- 공식 사이트 링크

### 선택 정보

- 성급
- 대표 이미지

## 11. SearchSummaryBar

### 역할

- 현재 검색 조건을 고정 표시한다.

### 포함 정보

- 호텔명
- 체크인 ~ 체크아웃
- 인원
- 객실 수
- 다시 검색 버튼

### 문구 예시

- `2026년 5월 10일 - 5월 12일, 성인 2명, 객실 1개`

## 12. EmptyState

### 역할

- 검색 결과가 없을 때 사용자 안내

### 제목 예시

- `검색 결과를 찾지 못했습니다`

### 본문 예시

- `날짜나 호텔명을 조정한 뒤 다시 검색해 보세요`

### 버튼

- `다시 검색`

## 13. ErrorState

### 역할

- 검색 실패 또는 공급처 수집 실패 안내

### 제목 예시

- `일부 요금을 불러오지 못했습니다`

### 본문 예시

- `현재 확인 가능한 공급처 기준으로 결과를 먼저 보여드립니다`

## 14. BrgGuidePanel

### 역할

- 사용자가 BRG 진행 전에 확인해야 할 항목을 안내한다.

### 권장 단계

1. 객실 조건 확인
2. 취소 정책 확인
3. 세금 포함 여부 확인
4. 외부 사이트 실제 총액 확인
5. 호텔 브랜드 정책 확인

### 문구 톤

- 설명형
- 확정형 금지
- 실무형 안내 유지

## 15. Footer

### 역할

- 운영 및 법적 정보 연결

### 필수 링크

- BRG란?
- 이용안내
- 면책사항
- 문의

### 하단 문구 예시

- `Ytravel은 숙박 요금 비교 및 BRG 탐색을 돕는 정보 서비스입니다`

## 16. 컴포넌트 우선순위

MVP 1차 필수

- SearchBar
- SearchSummaryBar
- BrgSummaryCard
- ProviderRateTable
- ProviderRateCard
- ConditionBadge
- MatchStatusPill
- EmptyState
- ErrorState

MVP 2차

- BrgGuidePanel
- HotelHeader
- Tooltip
- ProviderFilterBar

## 17. 구현 주의사항

- 한국어는 영어보다 줄바꿈이 쉽게 발생하므로 버튼 폭과 태그 길이를 넉넉히 잡는다.
- 가격 숫자는 한글 문장과 분리해 시각적 덩어리로 보여준다.
- 고급스러운 인상은 과한 장식보다 정리된 간격과 일관된 라벨에서 나온다.
