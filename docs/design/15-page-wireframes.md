# Ytravel 페이지별 와이어프레임

## 1. 목적

이 문서는 Ytravel의 주요 페이지를 화면 단위로 구체화한 와이어프레임 문서다.

목표

- 화면별 정보 우선순위를 고정한다.
- 디자인과 개발이 같은 구조를 기준으로 작업하게 한다.
- 한국어 UI 문장 길이와 비교 서비스 특성을 반영한다.

## 2. 공통 레이아웃 규칙

### 헤더

- 좌측: Ytravel 로고
- 우측: `검색 시작`, `BRG란?`, `서비스 소개`

### 푸터

- `이용안내`
- `면책사항`
- `문의`
- `약관`

### 공통 섹션 원칙

1. 페이지 목적이 첫 화면에서 이해되어야 한다.
2. 핵심 행동은 첫 스크롤 전에 노출한다.
3. 비교 정보는 요약 후 상세 순으로 배열한다.
4. 면책 또는 안내 문구는 하단에 배치하되 숨기지 않는다.

## 3. 홈 `/`

### 페이지 목표

- Ytravel의 정체성을 설명한다.
- 사용자가 즉시 검색을 시작하게 만든다.

### 섹션 구성

1. 헤더
2. 히어로
3. BRG 설명
4. Ytravel 차별점
5. 비교 기준 설명
6. 하단 CTA
7. 푸터

### 와이어프레임

```text
+--------------------------------------------------------------------------------------------------+
| Ytravel                                              검색 시작   BRG란?   서비스 소개         |
+--------------------------------------------------------------------------------------------------+
|                                                                                                  |
|  공식 홈페이지보다 낮은 요금, 더 정교하게 확인하세요                                              |
|  Ytravel은 호텔 공식 홈페이지와 OTA 요금을 BRG 기준으로 비교합니다                            |
|                                                                                                  |
|  [ 호텔명 입력                               ] [ 체크인 ] [ 체크아웃 ] [ 검색하기 ]               |
|  [ 투숙 인원 ] [ 객실 수 ]                                                                       |
|                                                                                                  |
+--------------------------------------------------------------------------------------------------+
|  BRG는 이렇게 확인합니다                                                                          |
|  1. 공식 홈페이지 요금 확인   2. OTA 최저가 비교   3. 동일 조건 여부 검토                         |
+--------------------------------------------------------------------------------------------------+
|  Ytravel이 함께 보는 항목                                                                     |
|  객실 타입 | 취소 정책 | 조식 포함 | 세금 포함 | 결제 방식                                       |
+--------------------------------------------------------------------------------------------------+
|  현재 수집 기준 결과이며, 실제 예약 전 외부 사이트 조건을 다시 확인해 주세요                      |
+--------------------------------------------------------------------------------------------------+
```

### 핵심 UI 포인트

- 검색 폼이 히어로의 중심이어야 한다.
- 설명은 3블록 이하로 짧게 유지한다.
- 여행 감성 이미지보다 신뢰감 있는 배경과 타이포를 우선한다.

## 4. 검색 결과 `/search`

### 페이지 목표

- 사용자가 한눈에 공식가, OTA 최저가, BRG 가능성을 판단하게 만든다.

### 섹션 구성

1. 검색 요약 바
2. 호텔 요약
3. BRG 요약 카드
4. 공급처 비교 표
5. 조건 비교 요약
6. 상세 이동 영역
7. 면책 문구

### 와이어프레임

```text
+--------------------------------------------------------------------------------------------------+
| Grand Hyatt Seoul | 2026년 5월 10일 - 5월 12일 | 성인 2명 | 객실 1개             [ 다시 검색 ]   |
+--------------------------------------------------------------------------------------------------+
| 호텔명 / 도시 / 브랜드 / 공식 사이트 링크                                                         |
+--------------------------------------------------------------------------------------------------+
| [ 가능성 높음 ]                                                                                  |
| 공식 홈페이지            OTA 최저가               예상 BRG가                                      |
| ₩320,000                ₩268,000                ₩201,000                                        |
| 현재 수집 기준으로 OTA 최저가가 더 낮습니다                                                       |
+--------------------------------------------------------------------------------------------------+
| 공급처         총액         취소         조식         세금         결제         보기               |
| Official      ₩320,000     무료 취소    없음         포함         선결제       가격 보기           |
| Trip.com      ₩268,000     무료 취소    없음         포함         선결제       가격 보기           |
| Agoda         ₩279,000     무료 취소    없음         포함         선결제       가격 보기           |
| Booking.com   ₩291,000     무료 취소    없음         포함         선결제       가격 보기           |
+--------------------------------------------------------------------------------------------------+
| 조건 비교 요약                                                                                    |
| 객실명 유사 | 취소 정책 일치 | 세금 포함 동일 | 조식 조건 동일                                    |
+--------------------------------------------------------------------------------------------------+
| [ 상세 비교 보기 ]                                                                                |
+--------------------------------------------------------------------------------------------------+
| 표시된 가격은 수집 시점 기준이며 실제 예약 화면과 다를 수 있습니다                                |
+--------------------------------------------------------------------------------------------------+
```

### 핵심 UI 포인트

- BRG 요약 카드는 표보다 먼저 보여야 한다.
- 가격 비교 표는 데스크탑에서 가장 강한 정보 영역이어야 한다.
- 한국어 라벨 길이를 고려해 표 컬럼 수는 과도하게 늘리지 않는다.

## 6. 호텔 상세 `/hotel/[slug]`

- BRG 판단에 필요한 구체적인 비교 근거를 제공한다.

### 섹션 구성

1. 호텔 헤더
2. BRG 요약
3. 공급처별 상세 비교
4. 객실 조건 비교 표
5. BRG 가이드 패널
6. 외부 링크
7. 면책 문구

### 와이어프레임

```text
+--------------------------------------------------------------------------------------------------+
| Grand Hyatt Seoul                                                                                 |
| Seoul, KR | Hyatt | 공식 사이트 보기                                                              |
+--------------------------------------------------------------------------------------------------+
| BRG 요약                                                                                          |

+--------------------------------------------------------------------------------------------------+
| 공급처별 상세                                                                                     |

+--------------------------------------------------------------------------------------------------+
| 객실 조건 비교                                                                                    |

| 세금            포함                        포함                                                   |

| BRG 진행 전 확인 항목                                                                             |
```

### 핵심 UI 포인트

- 상세 페이지는 설명보다 비교 근거가 우선이다.
- 표는 시각적으로 정돈되어야 하며, 차이 항목은 별도 강조한다.
- 외부 링크는 카드 하단보다 비교 근거 다음에 배치한다.

## 7. BRG 안내 `/brg`

### 페이지 목표

- BRG의 개념, 조건, Ytravel의 역할을 명확히 설명한다.

### 섹션 구성

1. 소개 헤더
2. BRG 정의
3. 성립 조건
4. Ytravel 비교 기준
5. 주의사항
6. 검색 CTA

### 와이어프레임

```text
+--------------------------------------------------------------------------------------------------+
| BRG란 무엇인가요                                                                                  |
| Best Rate Guarantee는 공식 홈페이지보다 낮은 동일 조건 요금이 있을 때                             |
| 호텔 브랜드 정책에 따라 가격 조정 또는 추가 할인이 적용될 수 있는 제도입니다                      |
+--------------------------------------------------------------------------------------------------+
| BRG 성립에 자주 필요한 조건                                                                        |
| 동일 객실 | 동일 날짜 | 동일 인원 | 동일 취소 정책 | 동일 조식 조건 | 동일 총액 기준             |
+--------------------------------------------------------------------------------------------------+
| Ytravel은 아래 항목을 함께 비교합니다                                                          |
| 객실 타입 | 취소 정책 | 세금 포함 여부 | 조식 포함 여부 | 결제 방식                               |
+--------------------------------------------------------------------------------------------------+
| [ 검색 시작 ]                                                                                     |
+--------------------------------------------------------------------------------------------------+
```

### 핵심 UI 포인트

- 설명은 길게 늘이지 않고 블록 단위로 쪼갠다.
- 법적 오해를 줄이기 위해 `가능성`, `정책에 따라 다름` 같은 표현을 유지한다.

## 14. 반응형 전환 기준

### 데스크탑

- 비교표 중심
- 요약 카드와 표가 한 흐름으로 연결

### 태블릿

- 요약 카드는 유지
- 표는 축약하거나 2행 구조 허용

### 모바일

- 카드 스택 중심
- 표는 카드형으로 치환
- 버튼은 전체 너비 사용 가능

## 15. 페이지 제작 우선순위

MVP 1차

1. 홈
2. 검색 결과
3. 호텔 상세
4. BRG 안내
5. 면책 페이지

MVP 2차

1. 서비스 소개
2. 공급처 소개
3. 도움말

## 16. 한 줄 요약

Ytravel의 페이지 구조는 `검색 시작 -> BRG 요약 -> 공급처 비교 -> 상세 근거 -> 외부 이동` 흐름이 끊기지 않도록 설계해야 한다.
