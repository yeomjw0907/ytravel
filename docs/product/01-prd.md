# Ytravel PRD

## 1. 제품 개요

- 서비스명: Ytravel
- 제품 정의: 호텔 공식 홈페이지 가격과 OTA 가격을 같은 조건으로 비교해 BRG(Best Rate Guarantee) 클레임 가능성을 빠르게 찾게 해주는 서비스
- 핵심 가치: 단순 최저가 나열이 아니라 `공홈 vs OTA`를 BRG 관점으로 비교하고, 사용자가 바로 판단할 수 있도록 조건 차이와 예상 할인 결과를 함께 제공

## 2. 문제 정의

현재 사용자는 다음 과정을 수작업으로 반복한다.

- 호텔 공식 사이트 방문
- Trip.com, Agoda, Booking.com 등 OTA를 각각 방문
- 객실 타입, 환불 정책, 조식 포함 여부, 세금 포함 여부를 직접 비교
- 더 싼 가격을 찾았더라도 BRG 대상인지 직접 판단

이 과정은 느리고, 실수 가능성이 높으며, 특히 동일 조건 비교가 어렵다.

## 3. 목표

Ytravel의 MVP 목표는 아래 3가지를 한 화면에서 해결하는 것이다.

- 공식 홈페이지 가격 확인
- OTA별 가격 및 최저가 확인
- BRG 청구 가능성이 높은 후보 자동 표시

## 4. 타겟 사용자

- 해외 체인 호텔을 자주 예약하는 여행자
- 가격 비교에 익숙한 헤비 유저
- BRG 제도를 적극 활용하려는 사용자
- 출장 숙소 예약 담당자

## 5. 핵심 사용자 시나리오

1. 사용자가 호텔명과 체크인/체크아웃 날짜를 입력한다.
2. Ytravel이 공식 홈페이지와 OTA 가격을 비교한다.
3. 사용자는 최저가 공급처와 공식가 차이를 확인한다.
4. 시스템은 동일 조건 여부와 BRG 가능성을 표시한다.
5. 사용자는 더 확인할 링크를 눌러 실제 예약 또는 BRG 클레임 준비를 진행한다.

## 6. MVP 범위

포함

- 호텔명, 날짜, 인원 기반 검색
- 공식 홈페이지 가격 카드
- OTA 가격 리스트
- 최저가 표시
- BRG 예상 적용가 계산
- 동일 조건 여부 요약
- 공급처 링크 제공

제외

- 대규모 전 세계 호텔 커버리지
- 로그인/회원 기능
- 결제 기능
- 자동 BRG 청구 대행
- 스크린샷 자동 보관

## 7. 핵심 지표

- 검색 완료율
- 검색 후 결과 페이지 도달률
- BRG 후보가 있는 검색 비율
- 결과 카드 클릭률
- 특정 공급처 링크 클릭률

## 8. 차별화 포인트

- 단순 숙박 비교 서비스가 아니라 BRG 특화 서비스
- 가격 숫자만이 아니라 조건 일치 여부까지 함께 표시
- 공식가 대비 BRG 예상 결과를 바로 계산
- 실제 사용자가 바로 행동할 수 있도록 공급처 링크와 근거 구조 제공

## 9. 주요 리스크

- 사이트별 크롤링 난이도와 구조 변경
- 약관 및 봇 차단 이슈
- 객실명과 정책 불일치로 인한 비교 오류
- 세금/통화/지역별 정책 차이

## 10. 제품 원칙

- 가격 비교보다 조건 비교를 우선한다.
- 수집 성공률보다 결과 신뢰도를 우선한다.
- 확정 표현보다 가능성 표현을 사용한다.

권장 문구 예시

- `BRG 가능성 높음`
- `조건 검토 필요`
- `동일 조건 아님`
- `현재 수집 기준 최저가`
