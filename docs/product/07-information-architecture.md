# Ytravel 정보구조(IA)

## 1. 목적

이 문서는 Ytravel의 페이지 구조, URL 구조, 탐색 흐름, 주요 정보 계층을 정의한다.

목표

- 사용자가 검색에서 비교 결과까지 빠르게 도달하게 한다.
- BRG 판단에 필요한 정보가 일관된 순서로 노출되게 한다.
- 이후 개발 시 라우팅과 데이터 구조의 기준선 역할을 한다.

## 2. 제품 구조 개요

Ytravel은 크게 4개 레이어로 구성된다.

1. 유입 및 탐색 레이어
2. 검색 및 결과 레이어
3. 상세 비교 레이어
4. 지원 정보 레이어

## 3. 사이트맵

```text
/
|-- /about
|-- /brg
|-- /search
|   |-- ?hotel=...&checkIn=...&checkOut=...&adults=...&rooms=...
|-- /hotel/[slug]
|   |-- /hotel/[slug]/rates
|   |-- /hotel/[slug]/brg-guide
|-- /providers
|   |-- /providers/trip-com
|   |-- /providers/agoda
|   |-- /providers/booking-com
|-- /legal
|   |-- /legal/terms
|   |-- /legal/disclaimer
|-- /help
```

## 4. 페이지별 역할

### 4.1 `/`

역할

- 서비스 소개
- BRG 개념 설명
- 검색 시작점 제공

핵심 콘텐츠

- 검색 바
- Ytravel 설명
- BRG 동작 방식
- 신뢰/면책 요약

### 4.2 `/search`

역할

- 검색 결과 중심 페이지
- 공식가와 OTA 가격을 한 화면에서 비교

핵심 콘텐츠

- 검색 요약
- BRG 요약 카드
- 공급처별 가격 리스트 또는 테이블
- 조건 비교 정보

### 4.3 `/hotel/[slug]`

역할

- 특정 호텔의 상세 비교 및 BRG 판단 근거 제공

핵심 콘텐츠

- 호텔 정보
- 가격 비교 상세
- 객실 조건 비교
- BRG 가이드
- 외부 링크

### 4.4 `/brg`

역할

- BRG 제도 설명
- Ytravel이 어떤 기준으로 비교하는지 설명

### 4.5 `/providers/*`

역할

- 공급처별 특징 설명
- 수집 범위와 유의사항 안내

### 4.6 `/legal/*`

역할

- 면책 문구
- 서비스 이용 조건
- 가격 정확도 관련 안내

## 5. 전역 탐색 구조

헤더 기본 항목

- Ytravel 로고
- 검색 시작
- BRG란?
- 서비스 소개

푸터 기본 항목

- 이용안내
- 면책사항
- 문의
- 약관

## 6. 정보 우선순위 규칙

검색 결과 및 상세 페이지에서 정보는 항상 아래 순서로 보여준다.

1. 검색 컨텍스트
2. 공식가와 OTA 최저가
3. BRG 상태
4. 동일 조건 여부
5. 공급처별 세부 조건
6. 외부 이동 링크
7. 면책 및 참고 안내

## 7. URL 설계 원칙

- 사용자가 공유 가능한 검색 URL을 가진다.
- 검색 조건은 query parameter로 유지한다.
- 호텔 상세는 slug 기반으로 구성한다.
- 공급처/법적 페이지는 고정 라우트로 유지한다.

예시

- `/search?hotel=grand-hyatt-seoul&checkIn=2026-05-10&checkOut=2026-05-12&adults=2&rooms=1`
- `/hotel/grand-hyatt-seoul`

## 8. 탐색 원칙

- 어떤 페이지에서도 검색으로 다시 돌아갈 수 있어야 한다.
- 검색 결과에서 상세 비교로 1회 클릭 이동 가능해야 한다.
- 상세 페이지에서 외부 공급처 링크까지 1회 클릭 이동 가능해야 한다.
- 면책 문구는 별도 페이지와 인라인 요약 둘 다 제공한다.
